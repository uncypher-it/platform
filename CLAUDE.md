# Uncypher Self-Hosted Deployment

This directory is a self-contained deployment package. All services use pre-built Docker images from Docker Hub (`siddhsingh/uncypher:*`). No source code or build step.

## Service Map

| Service | Image | Port | Purpose |
|---------|-------|------|---------|
| traefik | traefik:v3.6 | 80, 443 | HTTPS ingress, routing, security middleware |
| egress-proxy | ubuntu/squid:6.6 | 3128 (internal) | Outbound traffic filtering for data residency |
| postgres | postgres:16 | 5432 (internal) | Primary database (`data_decipher`) |
| redis | redis:7-alpine | 6379 (internal) | Cache, Celery broker, event streams |
| backend | siddhsingh/uncypher:backend | 8000 (internal) | FastAPI REST API |
| celery-worker | siddhsingh/uncypher:backend | internal | Background task processor |
| celery-beat | siddhsingh/uncypher:backend | internal | Periodic task scheduler |
| chat-consumer | siddhsingh/uncypher:chat-consumer | internal | AI chat via Redis streams + OpenAI |
| frontend | siddhsingh/uncypher:frontend | 3000 (internal) | Next.js web UI |
| kb-postgres | postgres:16 | 5432 (internal) | Knowledge base database (`kb`) |
| kb-redis | falkordb/falkordb:edge | 6379 (internal) | Graph database |
| kb-session-redis | redis:7 | 6380 (internal) | KB session storage |
| kb-chroma | ghcr.io/chroma-core/chroma:0.5.23 | 8000 (internal) | Vector DB for KB embeddings |
| kb-elasticsearch | elasticsearch:8.11.0 | 9200 (internal) | Full-text search for KB |
| kb-api | siddhsingh/uncypher:kb-api | 8000 (internal) | Knowledge Base API |
| kb-frontend | siddhsingh/uncypher:kb-frontend | 3000 (internal) | Knowledge Base UI (Vite) |
| demo-postgres | postgres:16 | 5432 (internal) | Demo e-commerce database |

| codebase-codex-runner | siddhsingh/uncypher:codebase-codex-runner | 9100 (internal) | Codex CLI execution sidecar |
| codebase-claude-runner | siddhsingh/uncypher:codebase-claude-runner | 9000 (internal) | Claude Code CLI execution sidecar |

## Network Architecture

- **uncypher-internal** (bridge, `internal: true`): All services. No internet access.
- **uncypher-egress** (bridge): Only Traefik + Squid. Internet-facing.

Services needing external API access (backend, chat-consumer, codex-runner) route through `HTTP_PROXY=http://uncypher-egress-proxy:3128`. Allowed domains in `egress-proxy/allowed_domains.txt`.

## File Layout

```
docker-compose.yml          — service definitions
.env / .env.example         — all configuration
setup.sh                    — automated setup
traefik/dynamic/tls.yml     — static TLS cert config
egress-proxy/squid.conf     — proxy rules
egress-proxy/allowed_domains.txt — egress allowlist
kb-init-sql/*.sql           — KB database schema (auto-loaded on first boot)
certs/                      — place static TLS certs here
```

## Environment Variables

All service-internal URLs are computed inside `docker-compose.yml`. The customer only configures:

**Required:**
- `FRONTEND_DOMAIN`, `API_DOMAIN`, `KB_DOMAIN`, `CODEBASE_DOMAIN` — Traefik routing
- `POSTGRES_PASSWORD`, `KB_POSTGRES_PASSWORD` — database credentials
- `OPENAI_API_KEY` — AI chat agent
- `ACME_EMAIL` — Let's Encrypt (if using)

**Auto-generated:**
- `JWT_SECRET_KEY` — generated by `setup.sh` if set to `CHANGE_ME`

**Optional:**
- `GITHUB_CLIENT_ID/SECRET` — GitHub OAuth
- `ANTHROPIC_API_KEY` — Anthropic API key for Claude Code agent
- `DEFAULT_CODEBASE_AGENT` — `claude` (default) or `codex`
- `REDIS_PASSWORD` — Redis auth (empty = no auth)
- `ALLOWED_IP_RANGES` — IP allowlist (default: `0.0.0.0/0`)
- `RATE_LIMIT_AVERAGE/BURST` — rate limiting
- `TLS_CERT_RESOLVER` — `letsencrypt` (default) or empty for static certs
- `LOG_LEVEL` — `INFO` (default)

## Common Operations

```bash
# Start / stop
docker compose up -d
docker compose down

# View logs
docker compose logs -f backend
docker compose logs -f frontend

# Restart single service
docker compose restart backend

# Check health
docker compose ps

# Pull updated images
docker compose pull && docker compose up -d
```

## Common Issues

- **Services won't start**: Check `docker compose logs <service>` — usually missing env vars
- **Frontend blank page**: Check `NEXT_PUBLIC_API_URL` and `FRONTEND_DOMAIN` match
- **TLS errors**: For Let's Encrypt, ensure domains resolve to the server IP and ports 80/443 open
- **Egress blocked**: Check `egress-proxy/allowed_domains.txt` for missing domains
- **DB init fails**: KB init SQL runs only on first boot. To re-run: `docker compose down -v` (destroys data)
- **Codebase agent 401**: Set `ANTHROPIC_API_KEY` or `OPENAI_API_KEY` in `.env`, or run `docker compose run --rm codebase-claude-runner claude login` for OAuth
